# 卡牌游戏前端开发文档

## 目标
实现一个卡牌游戏前端：两名玩家各有 10 点生命值，每回合各自选择一张卡，在双方都选择完成后同时公布并结算。

## 卡牌与回合
每回合从三张卡中选择一张：
- 进攻
- 防守
- 休养

双方都选择结束后进行结算。

## 结算规则（HP 变化）
- 进攻 vs 进攻：双方 -2
- 进攻 vs 防守：进攻 0，防守 0
- 进攻 vs 休养：进攻 0，休养 -2
- 防守 vs 休养：防守 0，休养 +1
- 防守 vs 防守：双方 0
- 休养 vs 休养：双方 +1

## 结束条件
- 任一方生命值归零，游戏结束。
- 若进行满 10 回合，则比较双方生命值高低决定胜负。

## 界面设计
### 入口页面（创建/进入房间）
- 输入：房间号、玩家昵称。
- 按钮：创建房间、进入房间。
- 提示：基础校验提示（空房间号/空昵称）。

### 对战页面
- 展示信息：
  - 自己血量、敌人血量、当前回合数。
- 操作区域：
  - 三张卡按钮：进攻 / 防守 / 休养。
  - 选择后进入“等待对方选择”状态，按钮禁用。
- 状态区：
  - 等待对方选择（loading/提示文本）。
  - 结算结果展示：本回合双方出牌与血量变化。
  - 结算后进入下回合，恢复可选状态。

### 匹配页面
- 展示房间号与玩家昵称。
- 匹配中提示（如“正在匹配对手”）。
- 可返回大厅页面。

### 结果页面
- 胜利 / 失败 / 平局三种状态展示。
- 展示最终血量与回合结束原因（生命归零或 10 回合结束）。

## 目录结构与静态资源
- `src/pages/`：页面级组件（入口/对战/结果）。
- `src/components/`：通用 UI 组件（卡牌按钮、血量展示、状态提示等）。
- `src/styles/`：样式文件（全局与页面样式）。
- `src/assets/`：通过模块导入的静态资源（图片、图标等）。
- `public/`：无需打包处理的静态资源（原样输出）。

## 开发步骤
1. 项目基础：确认前端工程可启动，整理目录结构与静态资源放置位置。
2. 入口页面：完成创建/进入房间表单与基础校验提示。
3. 匹配页面：展示匹配中信息与返回大厅入口。
4. 对战页面框架：搭建基础布局与区域（血量、回合、卡牌区、状态区）。
5. 出牌与等待：实现卡牌选择状态切换与等待提示的展示逻辑。
6. 结算展示：展示本回合双方出牌与血量变化。
7. 回合推进：结算后进入下一回合并恢复可选状态。
8. 结果页面：根据胜负/平局状态展示最终结果与结束原因。
